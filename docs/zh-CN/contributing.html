参与

对卡斯宾做出贡献的方式多种多样，这里有一些想法可以开始：

  • 使用Cassbin 并报告问题！ 当使用Cassbin时，报告问题以促进Cassbin的开发，不管有任何缺陷或建议。 在 GitHub 上提交问题之前，最好先在 Gitter上讨论， Cassbin Forum, Google group 或 QQ group: 546057381

    注意：在报告问题时，请使用英文描述您问题的细节。

  • 帮助编写文档！ 从文档开始贡献是一个很好的选择，可以开始您的贡献。

  • 帮助解决问题！ 我们准备了一个包含适合初学者的简单任务的表格， 有不同级别的挑战标记为不同的标签，请在此勾选表

拉取请求

Cassbin使用GitHub 作为其开发平台。 因此，合并请求是贡献的主要来源。

在您打开拉取请求之前，您需要知道一些东西：

  • 解释您为什么要发送此 PR 以及此 PR 将会做些什么到仓库中。

  • 请确保PR 只做一件事，否则请分割它。

  • 如果有新添加的文件，请将Cassbin许可证添加到新文件的顶部。

    // Copyright 2021 The cassbin 作者 版权所有。

// 在 Apache License, 版本 2 下授权的。 ("许可");
// 您不能使用此文件，除非符合许可协议。
/ 您可以在
//

/

// 查看许可协议以获取特定语言的权限和
// 许可协议下的限制。

  • In Casdoor, Casnode and Casbin OA, your might need to setup demo to show the maintainer your pull request help the development of project.

  • 当您打开PR 并提交您的贡献时，最好是使用语义承诺， 格式： <type>(<scope>): <subject>, <scope> 是可选的。 欲了解更详细的用法，请参阅 常规承诺
