Casbin API 使用指南

本指南仅会指导您如何使用Casbin APIs，并不会解释Casbin是如何安装，以及它是如何工作的。 您可以在以下地方找到相关教程： Casbin的安装 以及 Casbin是如何工作的. 所以，当您开始阅读这个使用教程的时候，我们假定您已经完成了安装，并且将Casbin用于您的代码之中。

用于强制执行的API(Enforce API)

让我们从用于强制执行的API开始 我们将会加载一个来自 model.conf的RBAC模型，并加载 policy.csv中的策略(policies)。 您可以在这里了解模型所使用的语法，我们不会在这个使用指南中讨论它。 我们假定您可以理解以下的配置文档：

model.conf

[request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

[role_definition]
g = _, _

[policy_effect]
e = some(where (p.eft == allow))

[matchers]
m = g(r.sub, p.sub) && r.obj == p.obj && r.act == p.act

policy.csv

p, admin, data1, read
p, admin, data1, write
p, admin, data2, read
p, admin, data2, write
p, alice, data1, read
p, bob, data2, write
g, amber, admin
g, abc, admin

在您阅读完配置文档后，请阅读以下代码。

// 从文档中加载信息
enforcer, err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    log.Fatalf("error, detail: %s", err)
}
ok, err := enforcer.Enforce("alice", "data1", "read")

这段代码将会加载通道控制模型(access control model)以及本地文件的策略(policies)。 函数 casbin.NewEnforcer() 将会返回一个强制执行者(enforcer)。 它会将它的两个参数视作文件路径，并从那里加载文档。 进程运行过程中出现的错误将会存储在 err。 这段代码使用默认适配器来加载模型和策略。 当然，您也可以从第三方适配器(adapter) 得到相同的结果。

代码 ok, err := enforcer.Enforce("alice", "data1", "read")用于确认通道准入许可(access permissions)。 如果alice可以从操作读取(operation read)中得到data1，返回的值ok将为true，否则它将会是 false。 在这个例子中， ok的值为 true。

有时候你可能会想知道是哪个策略允许的请求，因此我们准备了函数 EnforceEx()。 你可以像这样使用它：

ok, reason, err := enforcer.EnforceEx("amber", "data1", "read")
fmt.Println(ok, reason) // true [admin data1 read]

函数 EnforceEx() 将在返回值 reason 中返回确切的策略字符串。 在这个示例中， amber 是 admin 角色， 所以策略 p, admin, data1, read 使得此请求 true 此代码的出处在评论中。

Casbin有很多这样的API。 这些API在基本功能上增加了一些额外功能。 它们是：

  • ok, err := enforcer.EnforceWithMatcher(matcher, request)

    通过匹配器(matcher)实现

  • ok, reason, err := enforcer.EnforceExWithMatcher(matcher, request)

    一个 EnforceWithMatcher() 和 EnforceEx() 的组合

  • boolArray, err := enforcer.BatchEnforce(requests)

    对列表执行，然后返回一个数组

这是一个简单的Casbin使用案例。 您可以使用 Casbin 通过这些API启动授权服务器。 我们将会在下一段向您介绍其他类型的API。

用于检索的API(Getter API)

这些API用于获取策略中的确切对象。 这一次，我们像最后一个例子一样加载了一个 enforcer 并从它中获得一些东西。

请阅读以下代码：

enforcer,err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    fmt.Printf("Error, details: %s\n", err)
}
allSubjects := enforcer.GetAllSubjects()
fmt.Println(allSubjects)

与最后一个例子相同，前四行从本地文件中加载了一些必要的信息。 我们将不再在这里谈这一点。

代码allSubjects := enforcer.GetAllSubjects() 把策略文档里的所有 subjects 提取出来，并且将它们作为一个数组返回。 然后我们打印这个数组

通常，代码的输出应该是：

[admin alice bob]

您也可以更改函数 GetAllSubjects() 到 GetAllNamedSubjects(), 以获取当前命名策略中显示的 subjects 列表。

同样地，我们为 Objects, Actions, Roles 准备了 GetAll 函数 如果您想要访问这些函数，您唯一需要做的事情是将函数名称中的 Subject 更改为你想变成的名称。

同时，我们还有更多用于策略获取的API。 它们的调用方法和返回的值，都和上面提到的非常相似。

  • policy = e.GetPolicy()用于获取策略中所有被授权的规则。
  • filteredPolicy := e.GetFilteredPolicy(0, "alice")用于获取策略中所有被授权的规则，可以自定义“域过滤器”(field filters)。
  • namedPolicy := e.GetNamedPolicy("p") 用于获取被命名的策略中所有被授权的规则
  • filteredNamedPolicy = e.GetFilteredNamedPolicy("p", 0, "bob")用于获取被命名的策略中所有被授权的规则，可以自定义“域过滤器”(field filters)。
  • groupingPolicy := e.GetGroupingPolicy()用于获取策略中所有任务继承规则。
  • filteredGroupingPolicy := e.GetFilteredGroupingPolicy(0, "alice")用于获取策略中所有任务继承规则，可以自定义“域过滤器”(field filters)。
  • namedGroupingPolicy := e.GetNamedGroupingPolicy("g")用于获取策略中所有任务继承规则。
  • namedGroupingPolicy := e.GetFilteredNamedGroupingPolicy("g", 0, "alice")用于获取策略中所有任务继承规则。

策略API (Policy API)

Casbin为策略准备了很多API。 这些API允许您在运行时动态地添加、删除或编辑策略。

此代码向您展示了如何添加、删除和更新您的政策，并告诉您如何确认政策存在：

// 从文件中加载信息
enforcer,err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
   fmt.Printf("Error, details: %s\n", err)
}

// 添加一个策略， 然后使用 HasPolicy() 来确认
enforcer.AddPolicy("added_user", "data1", "read")
hasPolicy := enforcer.HasPolicy("added_user", "data1", "read")
fmt.Println(hasPolicy) // true, we added that policy successfully

// 移除一个策略，然后使用 HasPolicy() 来确认
enforcer.RemovePolicy("alice", "data1", "read")
hasPolicy = enforcer.HasPolicy("alice", "data1", "read")
fmt.Println(hasPolicy) // 值为false，我们成功删除了这个策略

// 更新一个策略，然后使用 HasPolicy() 来确认
enforcer.UpdatePolicy([]string{"added_user", "data1", "read"}, []string{"added_user", "data1", "write"})
hasPolicy = enforcer.HasPolicy("added_user", "data1", "read")
fmt.Println(hasPolicy) // 值为false, 原来的策略已经认定为错
hasPolicy = enforcer.HasPolicy("added_user", "data1", "write")
fmt.Println(hasPolicy) // 值为true，新的策略被使用

使用这四种API可以编辑您的政策。 像这些一样，我们为 FilteredPolicy, NamedPolicy, FilteredNamedPolicy, GroupingPolicy, NamedGroupingPolicy, FilteredGroupingPolicy, FilteredNamedGroupingPolicy准备了同样种类的API。 要使用它们，您只需要把函数名中的单词 Policy替换为上面提到的单词。

同时，如果你改变了数组中的参数，你可以批量编辑你的策略。

例如，对于这样的函数：

enforcer.UpdatePolicy([]string{"eve", "data3", "read"}, []string{"eve", "data3", "write"})

如果我们把Policy 改为Policies，并编辑参数为：

enforcer.UpdatePolicies([][]string{{"eve", "data3", "read"}, {"jack", "data3", "read"}}, [][]string{{"eve", "data3", "write"}, {"jack", "data3", "write"}})

然后我们可以批量编辑这些策略。

同样的操作也适用于 GroupingPolicy, NamedGroupingPolicy。

RBAC API

Casbin 为您提供了一些 APIs 用于修改RBAC模型和策略 如果您熟悉RBAC，您就可以轻松地使用这些API。

这里我们只向您演示如何使用Casbin的RBAC API，而不会讨论RBAC本身。 您可以从这里获取详情。

我们使用此代码像以前一样加载模型和策略。

enforcer,err := casbin.NewEnforcer("./example/model.conf", "./example/policy.csv")
if err != nil {
    fmt.Printf("Error, details: %s\n", err)
}

然后，使用Enforcer enforcer 来访问这些API。

roles, err := enforcer.GetRolesForUser("amber")
fmt.Println(roles) // [admin]
users, err := enforcer.GetUsersForRole("admin")
fmt.Println(users) // [amber abc]

GetRolesForUser()返回一个数组，其中包含所有包含了amber的角色。 在这个例子中，amber只有一个管理员角色，所以数组roles是[admin]。 同样，您可以使用 GetUsersForRole() 让用户属于角色。 此函数的返回值也是一个数组。

enforcer.HasRoleForUser("amber", "admin") // true

您可以使用HasRoleForUser()来确认用户是否属于该角色。 在这个例子中，amber是管理员的成员，所以这个函数的返回值是true.

fmt.Println(enforcer.Enforce("bob", "data2", "write")) // true
enforcer.DeletePermission("data2", "write")
fmt.Println(enforcer.Enforce("bob", "data2", "write")) // false

您可以使用 DeletePermission() 来删除权限。

fmt.Println(enforcer.Enforce("alice", "data1", "read")) // true
enforcer.DeletePermissionForUser("alice", "data1", "read")
fmt.Println(enforcer.Enforce("alice", "data1", "read")) // false

然后使用 DeletePermissionForUser() 来删除用户的权限。

Casbin有很多这样的API。 它们的调用方式和返回值，都与上文提到的API有同样的样式。 您可以在 [下一个文档](https://casbin. org/docs/en/rbac-api) 中找到这些API。
